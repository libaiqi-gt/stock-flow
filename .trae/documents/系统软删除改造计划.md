我将分步实施软删除改造，涉及数据库模型变更、DAO层逻辑重构以及API文档更新。

### 1. 数据库模型变更 (Models)
为以下实体结构体添加 `IsDeleted` (bool) 和 `DeletedAt` (*time.Time) 字段，并配置 GORM 索引：
- `internal/models/user.go`
- `internal/models/material.go`
- `internal/models/inventory.go`
- `internal/models/outbound.go`

代码变更示例：
```go
type Material struct {
    // ... 原有字段
    IsDeleted bool       `gorm:"default:false;index" json:"is_deleted"` // 软删除标记
    DeletedAt *time.Time `json:"deleted_at"`                            // 删除时间
}
```

### 2. DAO 层逻辑重构 (Data Access)
修改 `internal/dao` 下的所有相关文件，替换物理删除逻辑并为查询添加过滤条件。

**2.1 修改删除操作 (Delete)**
将原有的 `DB.Delete()` 替换为更新操作：
- **涉及文件**: `inventory_dao.go`, `material_dao.go`
- **逻辑**: `DB.Model(&Entity{}).Where("id = ?", id).Updates(map[string]interface{}{"is_deleted": true, "deleted_at": time.Now()})`

**2.2 修改查询操作 (List/Get)**
为所有查询方法（List, GetByID, GetByCode等）添加 `is_deleted = false` 过滤条件。
- **涉及文件**: `inventory_dao.go`, `material_dao.go`, `user_dao.go`, `outbound_dao.go`
- **注意**: 对于 `InventoryDao.List` 中的 Join 查询，明确指定表名（如 `wms_inventory.is_deleted = 0`）以避免字段冲突。

### 3. API 文档与注释更新
- 更新 Controller 层中 Delete 接口的 Swagger 注释，说明该接口执行软删除。
- 确保 API 文档反映最新的业务逻辑。
