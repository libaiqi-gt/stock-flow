我将实现物料基础信息的批量导入功能，基于您提供的Excel模板格式。

### 1. 定义数据结构 (DTO)
我将复用 `InventoryService` 中定义的 `BatchImportResult` 结构，或者在 `MaterialService` 中定义一个新的。考虑到代码复用，我会在 `MaterialService` 中使用相同的结构（因为它们在同一个 `services` 包下）。

### 2. 实现 Service 层逻辑
在 `internal/services/material_service.go` 中添加 `BatchImport` 方法：
- **Excel解析**：使用 `excelize` 库读取上传的文件。
- **字段映射**：
    - 物料编号 -> Code
    - 物料名称 -> Name
    - 物料类型 -> Category
    - 规格 -> Spec
    - 单位 -> Unit
    - 厂家/品牌 -> Brand
    - 安全库存 -> SafetyStock (int64)
    - 有效期报警时限/天 -> ExpiryAlertDays (int)
    - 开封效期/天 -> OpenedExpiryDays (int, 非必填)
- **业务逻辑**：
    - 逐行校验必填字段。
    - 检查物料编号是否已存在（如存在则跳过并记录错误，或者根据需求覆盖，通常批量导入新增默认为跳过已存在的）。
    - 调用 `materialDao.Create` 插入数据。
    - 汇总成功/失败数量及错误信息。

### 3. 实现 Controller 层逻辑
在 `internal/controllers/material_controller.go` 中添加 `BatchImport` 方法：
- 接收 `multipart/form-data` 文件上传。
- 校验文件格式和大小。
- 调用 Service 层处理。
- 返回导入结果。

### 4. 注册路由
在 `internal/routers/router.go` 中注册路由：
- `POST /api/v1/materials/import` (通常需要管理员权限)
